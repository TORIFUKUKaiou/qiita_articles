# はじめに

[Codex](https://platform.openai.com/docs/codex/overview)（OpenAIが社内で活用しているソフトウェアエージェント実行環境）でElixirを動かしてみたら、意外とサクッとインストールできた。が、`mix deps.get`で詰まった。そんな実験ログと気づきをメモとして残しておきます。

## Codexとは？

OpenAI社が提供している、クラウド上で動くソフトウェア開発エージェント。環境としてはDockerコンテナであり、いくつか特徴がある：

* Dockerイメージは、2025-06-29現在[codex-universal](https://github.com/openai/codex-universal)固定（将来的には他のものも設定できそうなUIの予感。あくまでも予感です）
* Setup script（初期セットアップ時）にはインターネットアクセスあり
* Agent phase（実行時）はデフォルトでネットアクセス不可（設定次第では一部許可可）
* セキュリティと濫用防止のため、**全通信がHTTP/HTTPSプロキシを経由**
* 環境変数として `http_proxy`, `https_proxy`, `CODEX_PROXY_CERT` などがあらかじめ設定済み

ざっくり言うと、アクセス許可を与えたGitHubリポジトリに対して、自然言語で指示を与えておくと、`git clone`して、コンテナ上でエージェント（代理人）が開発をしてくれます。

# ElixirとErlangの実行環境を整える

[codex-universal](https://github.com/openai/codex-universal)イメージにElixirはインストールされていません。そこでセットアップスクリプトを設定します。ここはできました。結論を先取りしておくと、`mix deps.get`は未だできていません。

[codex-universal](https://github.com/openai/codex-universal)イメージで用意されているものを貼っておきます。(2025-06-29)

![スクリーンショット 2025-06-29 18.44.54.png](https://qiita-image-store.s3.ap-northeast-1.amazonaws.com/0/131808/6b53514b-ff00-46f4-85bb-deeceb2c7ad2.png)



## ElixirとErlangのインストール

[Phoenix Express](https://hexdocs.pm/phoenix/up_and_running.html#phoenix-express)というPhoenixアプリ開発に必要なものを一通り揃えてくれるツールがあります。その中で使われているElixirの[インストールスクリプト](https://elixir-lang.org/install.sh)を使うと、ElixirとErlangのインストールは割とすんなりとできました :tada::tada::tada: 

```bash
curl -fsSo ./install.sh https://elixir-lang.org/install.sh
chmod +x ./install.sh

./install.sh elixir@1.18.4 otp@27.3.4.1

echo 'export PATH=$HOME/.elixir-install/installs/otp/27.3.4.1/bin:$PATH' >> ~/.bashrc
echo 'export PATH=$HOME/.elixir-install/installs/elixir/1.18.4-otp-27/bin:$PATH' >> ~/.bashrc

export PATH=$HOME/.elixir-install/installs/otp/27.3.4.1/bin:$PATH
export PATH=$HOME/.elixir-install/installs/elixir/1.18.4-otp-27/bin:$PATH
```

他の手段でも、ElixirとErlangのインストールはできると思います。[https://elixir-lang.org/install.sh](https://elixir-lang.org/install.sh)を使う例を示しておきます。Agent phase（実行時）で`elixir`コマンドや`iex`コマンド、`mix`コマンドが使えることは確認しました。  

ここは問題ありません。これは嬉しいです。夢が広がります。

## `mix deps.get`ができない

しかし、`mix deps.get`ができておりません。  
Pythonで言うところの`pip install`やRubyの`bundle install`、Nodeの`npm install`に相当する、外部ライブラリのセットアップコマンドです。  

### mix local.hex

`mix deps.get`の前に、`mix local.hex`をしておきます。  


```bash
mix local.hex --force
```

エラー内容を貼っておきます。  

```elixir
** (Mix) httpc request failed with: {:bad_status_code, 503}

Could not install Hex because Mix could not download metadata at https://builds.hex.pm/installs/hex.csv.

Alternatively, you can compile and install Hex directly with this command:

    $ mix archive.install github hexpm/hex branch latest
```

これは `mix archive.install github hexpm/hex --force` に切り替えて成功します。  
ここが重要なところで、GitHubと通信する場合はうまくいきます。

### 問題発生：`mix deps.get` で失敗

いよいよ`mix deps.get`で依存を取得することを目指してみます。しかしここでつまずきます。

```bash

```

## 原因の考察

Codexのドキュメントにこんな記述が：

> All outbound internet traffic passes through this proxy.

> Codex installs a proxy certificate into the system trust store. This certificate's path is available as the environment variable `$CODEX_PROXY_CERT`.

つまりSSLの証明書検証でコケてる。curlやpipは `$CODEX_PROXY_CERT` を参照してOKを出すようになっているが、Elixir（というかErlang VM）はそこまでやさしくありません。  

> おそらく、`repo.hex.pm` へのTLS通信時に CODEX_PROXY_CERT に対応するCAが正しく反映されていない（または Erlang/OTP の :certifi 相当が参照できていない）ことが原因では？
